Plugin SDK

Build plugins for MacroPad in two files. Your plugin gets a full SDK for running shell commands, persisting data, making HTTP requests, and pushing events to connected phones.

Overview

MacroPad plugins are Node.js modules that live in userData/plugins/<id>/. Each plugin exposes a map of action handlers — functions that fire when a button configured to that action is pressed on the phone.

Plugins run inside the same Electron process as the server, with access to Node.js built-ins and the MacroPad SDK. They are loaded once at startup and hot-reloaded on demand without restarting the app.

Quick start

Create a folder with two files:

my-plugin/
  manifest.json
  index.js

Minimal manifest.json:

{
  "id":      "my-plugin",
  "name":    "My Plugin",
  "version": "1.0.0",
  "actions": [
    { "key": "my-plugin.doThing", "label": "Do thing" }
  ]
}

Minimal index.js — factory pattern (recommended):

// index.js
module.exports = (sdk) => ({

  'my-plugin.doThing': async (params) => {
    const out = await sdk.shell.execAsync('date')
    sdk.log.info('Current date: ' + out)
  }

})

Load it in the app: open the Plugin Marketplace → Developer tab → "Load from folder".

File structure

my-plugin/
  manifest.json   ← required: metadata, actions list, pricing
  index.js        ← required: action handlers
  icon.png        ← optional: shown in marketplace (128×128)
  lib/            ← optional: helper modules (require('./lib/...'))
  assets/         ← optional: any other files your plugin needs
Subdirectories are fully supported — both when loading from folder and when installed via zip from the marketplace.

manifest.json

Full reference of every supported field:

FieldTypeDescription
idstringrequired Unique slug. Must match the folder name. Use kebab-case.
namestringrequired Human-readable name shown in the marketplace.
versionstringrequired Semver string, e.g. "1.2.0". Used for update checks.
actionsarrayrequired List of action definitions (see below).
descriptionstringoptional Short description shown in the marketplace.
authorstringoptional Your name or handle.
authorUrlstringoptional Link to your website or GitHub profile.
homepagestringoptional Docs or repo URL. Shown as "↗ Docs" link in marketplace.
iconstringoptional Path to icon image relative to plugin folder, or a URL.
pricenumberoptional Price in cents. 0 = free. Set to e.g. 800 for $8.
purchaseUrlstringoptional Your payment link (Stripe, Gumroad, LemonSqueezy). Required when price > 0.

licensestringoptional e.g. "MIT"

Each item in the actions array:

FieldTypeDescription
keystringrequired Unique action identifier. Convention: <plugin-id>.<verb>
labelstringrequired Shown in the admin panel action dropdown.
paramsarrayoptional Configurable inputs rendered in the admin panel (see Plugin params section).

Param definition

FieldDescription
keyUsed as the key in params object passed to your handler

index.js patterns

Factory function (recommended)

Export a function that receives sdk and returns your handlers. This gives you full SDK access:

module.exports = (sdk) => ({

  'my-plugin.action': async (params) => {
    // params = values from admin panel inputs (if any)
    sdk.log.info('action fired!')
  }

})

Plain object (simple plugins)

For plugins that don't need the SDK — just export a plain object:

const { execSync } = require('child_process')

module.exports = {
  'my-plugin.action': (params) => {
    execSync('notify-send "hello"')
  }
}
Action handlers can be async. They have a 10-second timeout — if they take longer, a warning is logged and the promise is dropped. Long-running tasks should be fire-and-forget (no await).

sdk.shell

Run shell commands. Uses child_process under the hood.

sdk.shell.exec(command)

Synchronous. Returns stdout as a string. Throws on non-zero exit.

const out = sdk.shell.exec('cat /proc/loadavg')
sdk.log.info(out) // "0.52 0.58 0.59 2/847 12345"

sdk.shell.execAsync(command)

Async. Returns a Promise that resolves with stdout, rejects on non-zero exit.

const ip = await sdk.shell.execAsync('hostname -I | awk \'{print $1}\'')
sdk.log.info('Local IP: ' + ip.trim())
Avoid using shell.exec/execAsync with raw user input — it runs in a shell. Use execFileSync/execFile from Node's child_process when you need to pass user-supplied arguments safely.

sdk.storage

Per-plugin persistent key/value storage. Data is saved to userData/plugins-data/<id>.json. Survives plugin reload and app restart. Uninstalling the plugin does not wipe the data file.

Methods

MethodDescription
storage.get(key)Returns the stored value, or undefined if not set.
storage.set(key, value)Stores any JSON-serialisable value.
storage.delete(key)Removes the key.
storage.clear()Wipes all data for this plugin.
// Counter that persists across button presses
'my-plugin.count': async () => {
  const n = (sdk.storage.get('count') || 0) + 1
  sdk.storage.set('count', n)
  sdk.log.info(`Pressed ${n} times`)
}

sdk.http

HTTP helpers with a 15-second timeout. Responses are parsed as JSON by default.

Methods

MethodDescription
http.get(url, headers?)GET request. Returns parsed JSON body.
http.post(url, body, headers?)POST request. body is serialised to JSON. Returns parsed JSON.
http.request(url, options)Full control. Options: method, headers, body, timeout.
// GET JSON
const data = await sdk.http.get('https://api.example.com/status')
sdk.log.info(data.status)

// POST JSON
await sdk.http.post('https://webhook.site/my-hook', { event: 'button_pressed' })

// Custom request
const res = await sdk.http.request('https://api.example.com/data', {
  method: 'PUT',
  headers: { 'Authorization': 'Bearer token123' },
  body: 'raw body',
  timeout: 5000
})

sdk.broadcast

Send a custom event to all connected phone clients over WebSocket. Use this to update the phone UI in real time from your plugin.

sdk.broadcast({ event: 'statusUpdate', value: 'online', color: '#4ade80' })

On the phone (PWA), listen in your own injected JS or a future plugin UI hook:

// The WebSocket message shape:
{
  "type": "pluginEvent",
  "pluginId": "my-plugin",
  "event": "statusUpdate",
  "value": "online",
  "color": "#4ade80"
}

sdk.log

Prefixed logging to the Electron console (visible in dev tools or terminal).

MethodDescription
log.info(...args)console.log with plugin prefix
log.warn(...args)console.warn with plugin prefix
log.error(...args)console.error with plugin prefix
sdk.log.info('Plugin started')
sdk.log.warn('Retrying...')
sdk.log.error('Failed:', err.message)

Plugin params (per-button config)

Declare a params array on any action to let users configure values per button in the admin panel. When a button is pressed, the configured values arrive in your handler as the params object.

// manifest.json
"actions": [
  {
    "key": "my-plugin.greet",
    "label": "Show greeting",
    "params": [
      {
        "key":         "name",
        "label":       "Person to greet",
        "type":        "text",
        "placeholder": "Alice"
      },
      {
        "key":     "delay",
        "label":   "Delay (ms)",
        "type":    "number",
        "default": 500
      }
    ]
  }
]
// index.js — params arrive in the handler
module.exports = (sdk) => ({
  'my-plugin.greet': async (params) => {
    const name  = params?.name  || 'World'
    const delay = params?.delay || 500
    await new Promise(r => setTimeout(r, delay))
    sdk.log.info(`Hello, ${name}!`)
  }
})

The user fills in the inputs when configuring the button. Each button can have different param values for the same action.

Publish to marketplace

  1. Create a GitHub release (or any public host) and upload a .zip of your plugin folder.
  2. Set downloadUrl in your manifest to the direct download URL of the zip.
  3. Fork chunkies/macropad and add your plugin to registry/registry.json.
  4. Open a Pull Request. Once merged, your plugin appears in the marketplace for all users.

registry.json entry format

{
  "id":             "my-plugin",
  "name":           "My Plugin",
  "version":        "1.0.0",
  "description":   "What it does",
  "author":         "Your Name",
  "authorUrl":      "https://github.com/you",
  "homepage":       "https://github.com/you/my-plugin",
  "icon":           "",
  "price":          0,
  "purchaseUrl":    "",
  "downloadUrl":    "https://github.com/you/my-plugin/releases/download/v1.0.0/my-plugin.zip",
  "license":        "MIT",
  "tags":           ["productivity"],
  "downloads":      0,
  "minAppVersion":  "1.0.0"
}
Your zip can use GitHub archive format — a single top-level folder wrapping your plugin files. The installer automatically flattens it.

Local development

You don't need to publish to test your plugin. In the marketplace:

  1. Go to the Developer tab.
  2. Click "📂 Load from folder" and select your plugin directory.
  3. The plugin is copied to userData/plugins/ and loaded immediately.
  4. Edit your index.js, then click "⟳ Reload plugins" in the admin panel — no restart needed.
Plugins loaded this way are marked _local: true and skipped during update checks.

Paid plugins

Set price (in cents) and purchaseUrl in your manifest. The app shows a "Buy $X →" button that opens the user's browser to your URL. You keep 100% of revenue — use any payment provider (Stripe, Gumroad, LemonSqueezy, Paddle).

{
  "price":       800,
  "purchaseUrl": "https://buy.stripe.com/your-link"
}

After purchase, direct the user to download the zip and load it manually (Developer tab), or provide a private downloadUrl in your delivery email.

Example: Basic plugin

A minimal plugin that sends a desktop notification:

// manifest.json
{
  "id":      "hello-world",
  "name":    "Hello World",
  "version": "1.0.0",
  "actions": [
    { "key": "hello-world.notify", "label": "Send notification" }
  ]
}
// index.js
const { execSync } = require('child_process')

module.exports = {
  'hello-world.notify': () => {
    execSync('notify-send "MacroPad" "Hello from your macro pad!"')
  }
}

Example: Text Macro plugin

Types a configurable text snippet, pauses, then fires a hotkey — all from one button press. This plugin demonstrates per-button params, cross-platform shell commands, and the SDK log.

// manifest.json (relevant section)
"actions": [{
  "key":    "text-macro.run",
  "label":  "Type text + hotkey",
  "params": [
    { "key": "text",   "label": "Text to type",              "type": "textarea" },
    { "key": "delay",  "label": "Pause before hotkey (ms)",  "type": "number", "default": 1000 },
    { "key": "hotkey", "label": "Hotkey after pause",         "type": "text", "placeholder": "ctrl+Return" }
  ]
}]
// index.js (simplified)
const { execFileSync } = require('child_process')
const os = require('os')

module.exports = (sdk) => ({
  'text-macro.run': async (params) => {
    const text   = params?.text   || ''
    const delay  = parseInt(params?.delay ?? 1000) || 0
    const hotkey = (params?.hotkey || '').trim()

    // Type the text
    if (text) {
      if (os.platform() === 'darwin') {
        execFileSync('osascript', ['-e', `tell application "System Events" to keystroke "${text}"`])
      } else {
        execFileSync('xdotool', ['type', '--clearmodifiers', '--delay', '20', '--', text])
      }
    }

    // Wait
    if (delay > 0) await new Promise(r => setTimeout(r, delay))

    // Fire the hotkey
    if (hotkey) {
      execFileSync('xdotool', ['key', '--clearmodifiers', hotkey])
    }
  }
})

Example: System Info plugin

Demonstrates every SDK method — shell, storage, http, broadcast, and log:

module.exports = (sdk) => ({

  // sdk.shell.execAsync
  'system-info.cpu': async () => {
    const raw = await sdk.shell.execAsync(
      "grep 'cpu ' /proc/stat | awk '{usage=($2+$4)*100/($2+$3+$4+$5)} END {printf \"%.1f\", usage}'"
    )
    sdk.broadcast({ event: 'cpu', value: raw.trim() })
  },

  // sdk.storage
  'system-info.count': () => {
    const n = (sdk.storage.get('presses') || 0) + 1
    sdk.storage.set('presses', n)
    sdk.log.info(`${n} presses so far`)
  },

  // sdk.http
  'system-info.ip': async () => {
    const data = await sdk.http.get('https://api.ipify.org?format=json')
    sdk.log.info('Public IP:', data.ip)
  }

})
