Build a plugin

Plugins live in userData/plugins/<id>/. Each plugin needs two files:

my-plugin/
  manifest.json   ← metadata, actions list, price
  index.js        ← action handlers (optionally using the SDK)

manifest.json

{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "What it does",
  "author": "Your Name",
  "authorUrl": "https://yoursite.com",
  "homepage": "https://github.com/you/my-plugin",
  "icon": "icon.png",
  "price": 0,
  "purchaseUrl": "https://buy.stripe.com/your-link",
  "downloadUrl": "https://github.com/you/my-plugin/releases/download/v1.0.0/my-plugin.zip",
  "license": "MIT",
  "tags": ["productivity"],
  "actions": [
    { "key": "my-plugin.doThing", "label": "Do Thing" }
  ]
}

Set price: 0 for free. For paid plugins, set your price in cents and add a purchaseUrl (Stripe, Gumroad, Lemon Squeezy — your choice, you keep the money).

index.js — with SDK

module.exports = (sdk) => ({

  'my-plugin.doThing': async (params) => {
    // Shell commands
    const out = await sdk.shell.execAsync('date')
    sdk.log.info('date output: ' + out)

    // Per-plugin storage
    const count = sdk.storage.get('count') || 0
    sdk.storage.set('count', count + 1)

    // HTTP requests
    const data = await sdk.http.get('https://api.example.com/data')

    // Send event to all connected phone clients
    sdk.broadcast({ event: 'myEvent', payload: data })
  }

})

Export a factory function that receives sdk for full API access. Or export a plain object for simple command-based plugins.

Publish to marketplace

Submit a PR to the stream-deck registry repo adding your plugin to registry.json. Your downloadUrl should point to a zip of your plugin folder.

Load local plugin

Pick a folder containing your manifest.json and index.js. It will be copied into the plugins directory and marked as a local dev plugin (skips update checks).
